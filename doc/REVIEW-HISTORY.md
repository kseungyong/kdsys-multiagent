# AI 검증 이력

> v1~v3까지 Gemini, GPT-4o에게 검증받은 전체 피드백 기록

---

## v1 검증 — Gemini 2.5 Flash (2026-03-30)

### 설계 내용
- 로컬 LLM 3개(Llama/Qwen/Gemma)로 자율 토론, 비용 $0
- 최종 결론만 클라우드 API로 검증

### Gemini 평가

**장점:**
- 획기적인 비용 절감 (핵심 가치)
- 무료 자율 토론으로 무제한 실험 가능
- 구조화된 메모리로 장기 학습 가능
- 데이터 프라이버시 이점

**단점:**
- 로컬 모델 품질 한계 (8B~14B는 GPT-4o/Claude 대비 현저히 낮음)
- 환각 증폭 위험 ("바보 3명이 더 큰 바보가 되는" 가능성)
- 토론 루프 (의미 없는 반복)
- Mac mini에서 3개 LLM 동시 운영 시 성능 저하

**핵심 제안:**
1. 동적 종료 조건 (고정 라운드 → 합의 감지)
2. 오케스트레이터 에이전트 (토론 조정자)
3. 벡터 DB (ChromaDB/LanceDB) + 로컬 임베딩
4. 피드백 루프 (클라우드 검증 → 다음 토론에 반영)
5. 전문 모델/MoE 탐색

**실현 가능성:** 기술적 높음, 실용적 가치 중간

## v2 검증 — GPT-4o (2026-03-30)

### v1 대비 변경점
- 오케스트레이터 에이전트 추가 (Gemma 4B)
- 동적 종료 조건
- 벡터 DB (LanceDB) + 시맨틱 검색
- 환각 방지 메커니즘
- 클라우드 피드백 루프

### GPT-4o 평가

**장점:**
- 오케스트레이터로 토론 품질 향상
- 동적 종료로 리소스 절약
- 시맨틱 메모리로 지식 연속성
- 자체 개선 구조

**단점:**
- 복잡성 증가 (유지보수 부담)
- 클라우드 의존성
- 로컬 모델 한계 여전

**빠뜨린 점:**
- 보안/프라이버시 계획
- 성능 모니터링 체계

**실현 가능성:** 8/10

**추가 제안:**
- 사용자 피드백 루프
- 리소스 최적화
- 백업/복구 프로세스

## v3 검증 — Gemini + GPT-4o 동시 (2026-03-30)

### v2 대비 핵심 변경
- "바보 2 + 천재 심판" 구조
- 로컬 모델 3→2명 (단순화)
- 오케스트레이터를 Claude Haiku로 교체 (품질 확보)
- 월 비용: ~$2 (v2의 $1 대비 $1 추가, 품질 2배)

### Gemini 평가 (9/10)

**장점:**
- 비용 효율성 극대화 (월 $2로 고품질)
- Claude 심판으로 환각 감지/루프 방지 비약적 향상
- 체계적 토론 관리 (동적 종료, 구조화된 결론)
- 자원 효율성 (토론자 2명으로 부하 감소)
- 장기 학습 가능 (집단 기억)

**단점:**
- Claude 의존성 (장애 시 전체 멈춤)
- 프롬프트 엔지니어링 복잡성
- 라운드별 API 레이턴시
- 라운드 요약 품질 위험
- 입력 토큰 비용 과소 계산 가능성
- 에스컬레이션 정책 모호

**"바보 2 + 천재 심판" 실효성:** "매우 실효성 높음"
- 인간 전문가 집단 의사결정 과정과 유사
- LLM의 "생성 vs 평가" 비용 차이를 잘 활용
- 심판 프롬프트의 정교함이 핵심

**단독 Claude보다 나은 조건:**
1. 비용 제약이 중요할 때
2. 다양한 관점/창의적 발상이 필요할 때
3. 시스템 학습/진화가 필요할 때
4. 로컬 모델 파인튜닝 가능할 때
5. 데이터 주권이 중요할 때

**추가 제안:**
- 입력 토큰 비용 정밀 분석
- 라운드 요약 품질 검증 프로세스
- 프롬프트 버전 관리 + A/B 테스트
- 동적 페르소나 전환
- 모니터링/알림 시스템

### GPT-4o 평가 (8/10)

**장점:**
- 비용 효율성
- Claude 심판으로 품질 관리
- 구조 단순화 (3→2명)
- 자체 개선 기능

**단점:**
- 외부 의존성 (Claude)
- 로컬 모델에 과도 의존 시 Claude에 떠넘기기
- UI 모드 복잡성

**빠뜨린 점:**
- 데이터 보안/프라이버시
- 성능 모니터링

**추가 제안:**
- 자동화된 성능 최적화 피드백 루프
- UI 간소화
- Claude 장애 대비 백업 프로세스

### 두 AI 공통 결론
- **구조 합격**, 보완점만 제시
- **오케스트레이터(Claude)가 핵심** — 프롬프트 품질이 전체 성패 좌우
- Claude 장애 대비 **로컬 폴백** 필요
- 입력 토큰 비용 **재계산** 필요
